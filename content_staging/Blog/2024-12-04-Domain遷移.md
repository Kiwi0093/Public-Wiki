# 前言

Gandi續費太貴了,所以我跳去Cloudflare了....

<!--truncate-->

# 原由

本來我的Domain是從[Freenom](https://www.freenom.com/en/index.html?lang=en)買的,後來因為該平台的問題被取消了`.com`網域的管理權限,然後就直接被轉去[Gandi](https://www.gandi.net/zh-Hant)
其實Gandi也是很大的DNS register服務也很不錯的,但是他的續費實在太貴了,續費一年要37USD左右...(我本來在Freenom的價格一年大概10USD)
所以我就遷移去了便宜很多的[Cloudflare](https://www.cloudflare.com/zh-tw/)

# Cloudflare DNS Register

Cloudflare是個很大的網路服務公司,很久以前我就把我的DNS設定到cloudflare去進行設定,免費的服務就很好用了

## Mail Alias

我現在Domain的信箱,是使用zoho的服務,不過老實說我對於印度人的印象沒那麼好...
Cloudflare其實是可以把Domain的Email alias到Gmail透過設定後除了可以收信以外也可以用自己的域名寄信

* [HalisUnreal的Blog](https://blog.halisunreal.com/posts/use-your-own-domain-email/)
* [使用 Cloudflare 在 Gmail 上配置域名邮箱](https://mskclover.com/2024/07/31/custom-email-on-gmail-by-cloudflare/)
* [使用 Cloudflare + Gmail 完成自定义域名邮箱收发](https://www.waterwater.moe/posts/2024/2024-01-01_%E4%BD%BF%E7%94%A8-cloudflare--gmail-%E5%AE%8C%E6%88%90%E8%87%AA%E5%AE%9A%E4%B9%89%E5%9F%9F%E5%90%8D%E9%82%AE%E7%AE%B1%E6%94%B6%E5%8F%91/)

可能後期還是要考慮一下是不是要為了縮減過多的服務平台把Email也轉移反正可以mapping到任意的Gmail帳號上隨便申請一個Gmail後拿來mapping就好了

這樣家人的mail address也容易弄好
